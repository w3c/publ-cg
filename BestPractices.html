This will be the temporary home to Best Practices documentation until a location and method for hosting and updating this material is agreed on.

Current Resources

There are a number of best practices documents and resources available for epub3.

The goal of this space is to pull together a list of these locations and documents in order to create a single universal resource for users.

Please add existing resources to pull from here:

  • DAISY: http://kb.daisy.org/publishing/docs/html/tables.html
  • DAISY: http://www.daisy.org/z3986-structure-guidelines
  • ePUB for education: https://w3c.github.io/publ-cg/education/epub-education.html#sec-overview
  • IEEE: http://ieee-adb.org
  • IDPF: https://idpf.github.io/epub-guides/package-metadata/
  • epub for education structural semantics: http://www.idpf.org/epub/profiles/edu/structure/
  • EBPAJ EPUB guideline: http://ebpaj.jp/counsel/guide

Roles

BISG

  • Help identify variations in market needs (higher ed versus trade, etc)
  • Gather existing best practices documentation
  • Identify pain points in implementation
  • Socialize best practices documentation as it is finalized to members

W3C Community Group

  • Cull recommendations from existing documentation
  • Test recommendations were necessary
  • Document results
  • Create, host, and manage final Best Practices artifact (location and style tbd)

Requests for Best Practices

Best practices requests have been made by the publishing community in the following fields:

  • CSS
    • css3 layout
    • column implementation
    • Fallbacks for some of the newer epub3 toolset (columns, flexbox)
    • a quality community developed "CSS reset" that guaranteed sufficient margins, good font size, etc. would be helpful
  • Accessibility
    • easy ways to implement accessibility
    • nuances of accessibility
    • how best to include the correct metadata pertaining to accessibility
    • trade publication accessibility
    • alt text best practices
    • video captions
    • proper heading structure
    • "practical" a11y
  • Cross-platform compatibility/interoperability of available ePub3 functionality: what works for iOS does not always work for Kindle.
  • Math display
  • Images
    • image sizing
    • complex image handling
    • resolution, size, file types
  • navigation
  • embedding fonts
  • characters/encoding
  • page numbering/pagelist
  • audio/video display
  • tables
  • margin notes/flowing in marginal elements
  • markup of non-structural headings (e.g. over blockquotes and lists)
  • use of aria-vs-epubtype attributes
  • proper use of @class, @role, and @data
  • proper use of rel
  • relationship of EPUB and schema.org
  • footnotes, endnotes, and marginal notes (up to all 3 in one document)
  • linking to ranges (sometimes extensive)
  • reference citations
  • inclusion of DOI for the EPUB
  • reader specific elements
  • language variety
  • advanced html5
